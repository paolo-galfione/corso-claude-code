# Screencast 1.2: Supervisione Umana Graduata

**Durata**: max 10 minuti
**Modulo**: 1 - Fondamenti
**Scopo**: Presentare il framework di supervisione che varia per ruolo

---

## SLIDE 1: Titolo

**Contenuto slide**:
```
SUPERVISIONE UMANA GRADUATA

Quanto controllo serve davvero?
```

**Testo da leggere**:
"Abbiamo visto che l'agente può lavorare autonomamente. Ma questo non significa che dobbiamo lasciarlo completamente libero. La chiave è calibrare la supervisione in base al contesto, al rischio e al ruolo. Vediamo come."

---

## SLIDE 2: Lo Spettro della Supervisione

**Contenuto slide**:
```
LO SPETTRO DELLA SUPERVISIONE

Alta                                              Bassa
──────────────────────────────────────────────────────
│         │              │              │            │
Approva   Revisiona      Monitora       Verifica    Trust
ogni      prima del      durante        dopo        completo
azione    commit         l'esecuzione   il fatto

     ◄─────────────────────────────────────────►
     Più sicuro                      Più efficiente
```

**Testo da leggere**:
"La supervisione è uno spettro. A un estremo approviamo ogni singola azione dell'agente. All'altro estremo, ci fidiamo completamente e verifichiamo solo il risultato finale. Nessun estremo è giusto o sbagliato: dipende dal contesto."

---

## SLIDE 3: Fattori che Influenzano la Supervisione

**Contenuto slide**:
```
FATTORI CHE INFLUENZANO LA SUPERVISIONE

┌─────────────────┬──────────────────┐
│ Supervisione +  │ Supervisione -   │
├─────────────────┼──────────────────┤
│ Codice critico  │ Codice isolato   │
│ Dati sensibili  │ Ambiente di test │
│ Prima volta     │ Task ripetitivo  │
│ Alta complessità│ Pattern noto     │
│ Irreversibile   │ Facilmente       │
│                 │ reversibile      │
└─────────────────┴──────────────────┘
```

**Testo da leggere**:
"Più supervisione quando lavoriamo su codice critico, dati sensibili, task nuovi, alta complessità o azioni irreversibili. Meno supervisione per codice isolato, ambienti di test, task ripetitivi, pattern consolidati e azioni facilmente reversibili."

Lo sviluppatore invece può delegare molto all'agente perché lavora su task ben definiti e verificabili automaticamente."

## SLIDE 5: Product Owner - Supervisione Alta

**Contenuto slide**:
```
PRODUCT OWNER

Flusso: Uomo → Agente

┌────────────────────────────────────────┐
│  "Claude, analizza le possibili        │
│   soluzioni per questo requisito"      │
│                                        │
│  Claude presenta opzioni               │
│                                        │
│  PO valuta e decide                    │
│                                        │
│  "Claude, documenta la soluzione       │
│   scelta con questi criteri"           │
└────────────────────────────────────────┘

L'agente informa, l'umano decide
```

**Testo da leggere**:
"Il Product Owner usa l'agente per esplorare soluzioni, analizzare trade-off, validare documentazione. Ma ogni decisione strategica passa dall'umano. L'agente informa, il PO decide. La supervisione è alta perché le conseguenze delle decisioni sono ampie."

---

## SLIDE 6: Sviluppatore - Supervisione Bassa

**Contenuto slide**:
```
SVILUPPATORE

Flusso: Agente ⟷ Uomo (supervisione)

┌────────────────────────────────────────┐
│  "Claude, implementa l'endpoint        │
│   seguendo le specifiche dell'analista"│
│                                        │
│  Claude implementa                     │
│  Claude esegue test                    │
│  Claude corregge errori                │
│  Claude verifica completezza           │
│                                        │
│  Dev revisiona risultato finale        │
└────────────────────────────────────────┘

L'agente esegue il ciclo completo
```

**Testo da leggere**:
"Lo sviluppatore può delegare molto di più. L'agente implementa, testa, corregge, verifica. Lo sviluppatore interviene solo per la review finale e per decisioni che richiedono contesto di business. La supervisione è bassa perché il lavoro è verificabile oggettivamente."

## SLIDE 7: Trust Incrementale

**Contenuto slide**:
```
TRUST INCREMENTALE

Settimana 1    ████████░░  Approva ogni azione
Settimana 2    ██████░░░░  Revisiona prima del commit
Settimana 4    ████░░░░░░  Monitora durante
Settimana 8    ██░░░░░░░░  Verifica dopo il fatto

┌─────────────────────────────────────────┐
│  Inizia con supervisione alta,          │
│  riducila man mano che acquisisci       │
│  confidenza con l'agente                │
└─────────────────────────────────────────┘
```

**Testo da leggere**:
"Il trust non è binario. Si costruisce nel tempo. Partite con supervisione alta: approvate ogni azione, capite come ragiona l'agente. Poi gradualmente riducete il controllo. Dopo qualche settimana saprete esattamente dove l'agente eccelle e dove ha bisogno di guida."

---

## SLIDE 8: Segnali di Allarme

**Contenuto slide**:
```
QUANDO AUMENTARE LA SUPERVISIONE

⚠ L'agente fa lo stesso errore due volte
⚠ Output inaspettato senza spiegazione
⚠ Task che richiede più tentativi del solito
⚠ Modifiche a file che non dovrebbe toccare
⚠ Comportamento diverso da quello richiesto

→ Non ignorare questi segnali
→ Aumenta supervisione temporaneamente
→ Analizza la causa e aggiusta il contesto
```

**Testo da leggere**:
"Ci sono segnali che indicano quando aumentare la supervisione. Se l'agente ripete errori, produce output inaspettati, o modifica file che non dovrebbe toccare, fermatevi. Aumentate temporaneamente la supervisione, capite cosa sta succedendo, e aggiustate il contesto nel CLAUDE.md o nelle istruzioni."

## SLIDE 9: Il Ruolo del CLAUDE.md

**Contenuto slide**:
```
CLAUDE.MD: IL CONTRATTO DI SUPERVISIONE

┌─────────────────────────────────────────┐
│  # Regole di Comportamento              │
│                                         │
│  - NON modificare file in /config       │
│  - SEMPRE eseguire test prima di commit │
│  - CHIEDERE conferma per delete         │
│  - MAI push diretto su main             │
│                                         │
└─────────────────────────────────────────┘

Le regole nel CLAUDE.md sono supervisione automatica
```

**Testo da leggere**:
"Il file CLAUDE.md è il vostro contratto di supervisione. Qui definite i limiti: cosa l'agente NON deve fare, cosa deve SEMPRE fare, quando deve chiedere conferma. È supervisione automatica che non richiede il vostro intervento costante."
