## SLIDE 4: MCP Filesystem

**Contenuto slide**:
```
MCP FILESYSTEM

Cosa permette:
- Leggere file fuori dal workspace
- Scrivere in directory esterne (con permessi)
- Accedere a log di sistema
- Leggere configurazioni globali

Esempio d'uso:
"Leggi il file di log dell'applicazione
 in /var/log/app/error.log e cerca
 errori correlati al bug che stiamo fixando"
```

**Testo da leggere**:
"MCP Filesystem estende l'accesso ai file. Normalmente Claude vede solo il workspace. Con questo server può leggere log di sistema, configurazioni globali, file in altre directory. Utile per debugging e analisi."

## DEMO 1: Configurare MCP SQLite

**Cosa mostrare**:

1. Mostrare la configurazione in `claude_desktop_config.json` o equivalente:

```json
{
  "mcpServers": {
    "sqlite": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-sqlite",
        "./database/northwind.db"
      ]
    }
  }
}
```

2. Riavviare Claude Code
3. Chiedere: "Mostrami le prime 5 righe della tabella Customers"
4. Claude usa MCP SQLite per fare la query

**Testo da leggere durante la demo**:
"Configuriamo MCP SQLite. Puntiamo al nostro database Northwind. Ora Claude può fare query dirette. Chiediamo i primi 5 customers e Claude interroga il database, non deve immaginare i dati."

## DEMO 2: Query di Analisi

**Cosa mostrare**:

1. Prompt: "Analizza la struttura del database Northwind. Mostrami le tabelle e le loro relazioni."

2. Claude usa MCP SQLite per:
   - Elencare le tabelle
   - Mostrare le foreign key
   - Descrivere le relazioni

3. Prompt follow-up: "Quanti ordini ha il customer ALFKI?"

4. Claude esegue la query e risponde con dati reali

**Testo da leggere durante la demo**:

## SLIDE 7: Configurazione Completa

**Contenuto slide**:
```
CONFIGURAZIONE COMPLETA

claude_desktop_config.json

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@anthropic/mcp-server-filesystem",
               "/var/log", "/etc/app"]
    },
    "sqlite": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-sqlite",
               "./database/northwind.db"]
    },
    "github": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-github"],
      "env": { "GITHUB_TOKEN": "..." }
    }
  }
}
```

**Testo da leggere**:
"Ecco la configurazione completa con tutti e tre i server. Ogni server ha il suo comando e i suoi argomenti. GitHub richiede un token per l'autenticazione. Una volta configurati, Claude li usa automaticamente quando servono."
