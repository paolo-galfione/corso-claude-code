# Cambio di Paradigma

Da Assistente ad Agente

<small>Claude Code per Team di Sviluppo</small>

---

## Il Modello "Assistente"

**Uomo: domanda → AI: risposta → Uomo: azione**

- L'uomo guida ogni passo
- L'AI suggerisce, l'uomo esegue
- Nessuna autonomia
- Nessuna verifica automatica

---
<!-- _class: diagram -->

## Il Modello "Agentico"

![Loop Agentico](./diagrammi/loop-agentico.svg)

---

## La Differenza Chiave

**Assistente**: "Ecco come potresti fare..."

**Agente**: "L'ho fatto. Ecco il risultato."

> L'agente si assume la responsabilità di verificare il proprio lavoro

---

## Perché Funziona

1. **Accesso agli strumenti**
   - File system, terminale, browser

2. **Contesto persistente**
   - Ricorda cosa ha fatto e perché

3. **Capacità di verifica**
   - Esegue test, legge output, interpreta errori

4. **Loop di feedback**
   - Ogni errore è informazione per migliorare

---

## Implicazioni per il Team

| Prima | Dopo |
|-------|------|
| Dev scrive codice | Dev supervisiona agente |
| Dev esegue test | Agente esegue test |
| Dev corregge bug | Agente corregge bug |
| Dev chiede review | Agente chiede conferma |

> Il ruolo umano si sposta verso supervisione, direzione, validazione

---
<!-- _class: takeaway -->

## Punti Chiave

- L'agente **agisce**, non suggerisce
- Il loop pianifica→esegue→verifica→corregge è **automatico**
- Gli errori sono opportunità di **auto-correzione**
- Il ruolo umano diventa **supervisione strategica**

*Prossimo: Supervisione Umana Graduata*
